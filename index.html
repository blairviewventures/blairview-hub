How to Store Your Hub and Use It Across Every Claude Session

Step-by-step. Never done GitHub before. Takes about 15 minutes total. After setup, updating takes 60 seconds per session.

What You're Building

  1. Part 1 — GitHub Pages: your hub at a permanent URL
  2. Part 2 — Using the hub to start any Claude session instantly
  3. Part 3 — Updating the hub after each session (60 seconds)
  4. Part 4 — The React artifact inside Claude (no GitHub needed)

Result: your full Blairview context lives at a permanent bookmarkable URL. Any new Claude session gets the full picture in 30 seconds. Everything stays current.

Part 1 — GitHub Pages Setup (one time, ~10 minutes)
Step 1 of 6
Create a free GitHub account
Go to: github.com → click Sign up

Use a personal email (not work). Username suggestion: giannaluppino or blairviewventures

Verify your email when prompted. That's it — free account, no credit card.
Note: GitHub is where developers store code. You're using it as free hosting for one HTML file. You don't need to understand anything else about it.
Step 2 of 6
Create a new repository (think of it as a folder)
Once logged in:

1. Click the + icon in the top-right corner → New repository
2. Repository name: blairview-hub (lowercase, hyphens, no spaces)
3. Description: "Blairview Ventures session hub" (optional)
4. Select Public (required for free GitHub Pages)
5. Check the box: Add a README file
6. Click Create repository
Why Public? Free GitHub Pages requires a public repo. The hub contains no sensitive information — all pricing, client names, and personal data should not be in this file. It's a context and strategy document only.
Step 3 of 6
Upload your hub file
You should now see your empty repository page.

1. Download BLAIRVIEW_SESSION_HUB.html from Claude to your computer
2. Rename the file on your computer to: index.html (this is important — it must be called index.html)
3. On the GitHub repo page, click Add file → Upload files
4. Drag index.html into the upload area (or click to browse)
5. Scroll down to Commit changes
6. Leave the default message ("Add files via upload") and click Commit changes
Step 4 of 6
Enable GitHub Pages (turns your file into a live website)
1. In your repository, click Settings (tab at the top, far right)
2. In the left sidebar, scroll down and click Pages
3. Under Source, select Deploy from a branch
4. Under Branch, select main and keep / (root)
5. Click Save

Wait 2–3 minutes. Refresh the page. You'll see a green box that says:
Your site is live at https://[yourusername].github.io/blairview-hub/
Bookmark that URL. That is your permanent hub address.
Step 5 of 6
Also upload your USER.md for Claude attachments
Back in your repository, click Add file → Upload files again.

Upload: USER_md_Gianna_Luppino.md (keep this filename as-is)

Commit it. It now lives at:
https://[yourusername].github.io/blairview-hub/USER_md_Gianna_Luppino.md
You can attach this file directly to any Claude conversation for full context.
Step 6 of 6
Test it
Open your hub URL in a browser. You should see the full session hub.

Bookmark it on your phone and computer. Label it: Blairview Hub

That's Part 1 done. Your hub is live.
Part 2 — Starting Any Claude Session Instantly

You now have two ways to give Claude full context in any new chat:

Method A — Paste the starter text (fastest, 10 seconds)

The USER.md has a "NEW CHAT STARTER" section at the top. Copy that text and paste it as your first message in any new Claude session. Claude will have the full brand, pricing, ICP, and lead context immediately.

The starter text is also in the React artifact hub (the "New Chat Starter" tab has a one-click copy button).

Method B — Attach the file (most complete)

When starting a new Claude chat, click the attachment (paperclip) icon and upload USER_md_Gianna_Luppino.md. Then say: "This is my full business context. Please read it before we begin."

This gives Claude everything — career arc, talk tracks by audience, all 10 revenue arms, hard limits, timing rules.

Method C — Link to the hub (when Claude has web access)

My session hub is at: https://[yourusername].github.io/blairview-hub/ Please read it for full context before we begin.
Best practice: Use Method A every session (it's instant). Use Method B for complex sessions where you want Claude to have the deep career detail. Method C when you want Claude to pull the absolute latest version.
Part 3 — Updating the Hub After Each Session (60 seconds)

At the end of any session where something significant changed — new leads, updated pricing, decisions made, files created — ask Claude:

"Update the session hub HTML and USER.md with everything we decided today. Give me the updated files to download."

Claude will generate updated versions. Then:

Update Step 1
Download the updated files from Claude
Download the updated index.html and/or USER_md_Gianna_Luppino.md to your computer.
Update Step 2
Replace the files on GitHub
1. Go to github.com/[yourusername]/blairview-hub
2. Click on index.html in the file list
3. Click the pencil icon (Edit this file) — top right of the file view
4. Select all the text (Cmd+A or Ctrl+A) and delete it
5. Paste in the new HTML content
6. Scroll down → click Commit changes

Live in 2 minutes. That's it.
Faster method: On the file page, click the pencil → then drag-and-drop the new file onto the editor. GitHub replaces the content automatically.
